What this is. Each vertical column is one byte from a classical hardware random source (your browser's crypto.getRandomValues). Each row is one bit of that byte, bright = 1, dark = 0. New columns scroll in from the right. This is representative — the live server-side QWR4U009 stream looks phenomenologically identical at this level.
What this is not. This page does not compute live p-values or flag anomalies per-moment. Doing so would create the signal channel the NT axiom of Generalized Quantum Theory forbids (see AMBIENT_GAMIFICATION.md). If you stare at this page and your entropy "looks different" when you focus — that's pareidolia. This is pure classical randomness by construction. No coupling to you is possible through this page.
why client-side entropy, not the server QRNG

Serving the server-side QWR4U009 stream live to an arbitrary browser would require a public WebSocket relay. We could build that, and the visual would be identical — but the instrument's job is not to expose its substrate live, it's to run detectors against it in background and surface anomalies as pings. A browser-local entropy visualization gives you the phenomenology of "watching entropy happen" without exposing any user-coupled surface that an optimizer could learn.

The raw stream used for the ambient pipeline is the MED1MQ16 at 125 KB/s on the Chronomancy server. Hardware details in AMBIENT.md. Calibration against a verified-null substrate is in calibration/2026-04-22_ensemble_v1.md.
